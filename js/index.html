‚öôÔ∏è How It Works

The test demonstrates the complete encryption workflow compatible with version 6.0's 4-byte header and HMAC authentication:

  1. PHP generates encryption parameters using the scrolling window mechanism
  2. JavaScript encrypts data using the same parameters
  3. HMAC authentication is calculated over the payload
  4. PHP verifies and decrypts the JavaScript-encrypted data

üì¶ Encrypted Data Format

Base64(HMAC-SHA256(32 bytes) + Header(4 bytes) + Ciphertext)

Where:

  • HMAC - 32-byte authentication tag
  • Header - 4-byte header with key/IV offsets for scrolling window
  • Ciphertext - AES-256-CBC encrypted data

üîí Security Features

  • 4-byte header - Enhanced scrolling window with independent offsets
  • HMAC-SHA256 - Authenticated encryption prevents tampering
  • Key derivation - Separate encryption and authentication keys
  • Encrypt-then-MAC - Industry-standard security pattern
  • Scrolling window - 8-byte window with 25 possible positions per key seed entry
  • ~40 million combinations - Enhanced key space from scrolling window mechanism

üìù Important Notes
